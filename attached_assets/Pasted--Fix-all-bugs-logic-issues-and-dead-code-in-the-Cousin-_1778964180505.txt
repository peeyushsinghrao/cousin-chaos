**🔴 CRITICAL — White screen after Onboarding (add this FIRST in the prompt)**

**Fix the blank white screen that appears after the Onboarding screen navigates to HomeScreen.**

The root cause is in `lib/core/theme/app_theme.dart`. The `pageTransitionsTheme` uses `FadeUpwardsPageTransitionsBuilder` for all platforms, but Flutter Web runs internally as the `linux` platform — and `FadeUpwardsPageTransitionsBuilder` starts its animation from a fully transparent/white state, causing a white flash that freezes visually.

**Fix it by replacing the entire `pageTransitionsTheme` block** in `AppTheme.darkTheme` with this:

```dart
pageTransitionsTheme: const PageTransitionsTheme(
  builders: {
    TargetPlatform.android: ZoomPageTransitionsBuilder(),
    TargetPlatform.iOS: CupertinoPageTransitionsBuilder(),
    TargetPlatform.linux: ZoomPageTransitionsBuilder(),
    TargetPlatform.macOS: ZoomPageTransitionsBuilder(),
    TargetPlatform.windows: ZoomPageTransitionsBuilder(),
    TargetPlatform.fuchsia: ZoomPageTransitionsBuilder(),
  },
),
```

`ZoomPageTransitionsBuilder` starts from a fully opaque/visible state so there is no white flash on any platform including web.

**Additionally**, in `lib/screens/splash/splash_screen.dart` and `lib/screens/onboarding/onboarding_screen.dart`, all `PageRouteBuilder` navigation calls use a `FadeTransition`. Make sure the `FadeTransition` starts from opacity `0.0` to `1.0` but the **background of the child is always `AppColors.background` (dark)**, never transparent white. Wrap the child in each `transitionsBuilder` with a `ColoredBox`:

```dart
transitionsBuilder: (context, animation, secondaryAnimation, child) {
  return FadeTransition(
    opacity: animation,
    child: ColoredBox(
      color: AppColors.background,
      child: child,
    ),
  );
},
```

Apply this same `ColoredBox` wrapper in:
- `SplashScreen._navigateToHome()` 
- `SplashScreen._navigateToNext()` (onboarding route)
- `OnboardingScreen` button `onPressed` navigation

This ensures that during the fade transition, the background is always dark — never the default white Material surface.
