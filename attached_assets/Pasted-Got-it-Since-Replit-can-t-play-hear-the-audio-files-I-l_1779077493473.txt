### 1. `wheel_spin.mp3`
**Used for:** Wheel Spin + Wheel Tick + Wheel Land (3 events, 1 file)

**What it sounds like:**
- Starts with a fast spinning/whooshing mechanical rotation sound
- In the middle it has rapid clicking tick sounds (like a roulette wheel ticking against pegs) that slow down gradually
- Ends with a final solid THUD or CLICK when the wheel stops on a player

**How to use in app:**
- Play from the **beginning** when the wheel starts spinning
- The ticking in the middle plays automatically as the wheel slows
- The final thud at the end = wheel land event
- Play the **full file** in one go — do NOT split it
- Should loop if the spin animation is longer than the sound

### 2. `tap.mp3`
**Used for:** Button Tap + Next Player button

**What it sounds like:**
- Very short, sharp, clean tap sound
- Like tapping a finger on a glass surface or a soft UI click
- Duration: under 0.3 seconds
- No echo, no reverb — very crisp and instant

**How to use in app:**
- Play on every button press across the whole app
- Play when "Next Player" is tapped
- Should never overlap — if tapped fast, restart the sound each time
- Volume should be medium-low so it doesn't overpower

### 5. `countdown.mp3`
**Used for:** Countdown timer (3-2-1 style)

**What it sounds like:**
- A series of 3 distinct beeps, each one getting slightly more intense/higher
- Like a classic 3-2-1 countdown — beep… beep… BEEP
- Final beep is louder or different tone to signal "GO"
- Duration: roughly 3 seconds total

**How to use in app:**
- Play when a timed phase is about to begin (e.g. alibi time, act it out timer starting)
- Do NOT loop — plays once before the timer starts
- The final beep should align with when the game action starts

### 6. `timer_tick.mp3`
**Used for:** Timer Tick (every second) + Timer End (when time runs out)

**What it sounds like:**
- Starts with a steady rhythmic tick sound (like a clock ticking)
- Each tick is short and precise — one tick per second
- Near the end it gets faster or more urgent
- Ends with a loud BUZZ or ALARM sound signalling time is up

**How to use in app:**
- Play one tick sound every second while timer is running
- When timer hits 0, play the final buzz/alarm portion of the file
- For the per-second tick: play only the first 0.5 seconds of the file on each tick
- For the end alarm: play the tail end of the file when `_timeLeft == 0`
- Alternatively play the whole file once and let it run — it covers both naturally

## ⚙️ Implementation Rules for Agent

```
1. All files go in:  assets/sounds/

2. Declare ALL files in pubspec.yaml under flutter > assets

3. Use a single AudioService / SoundManager class —
   never call AudioPlayer directly from UI widgets

4. Every sound must have a VOLUME level set:
   - tap.mp3          → 0.5 (subtle)
   - card-flip.mp3    → 0.7
   - card_reveal.mp3  → 0.8
   - countdown.mp3    → 0.9
   - timer_tick.mp3   → 0.6
   - wheel_spin.mp3   → 0.9
   - win.mp3          → 1.0 (full)
   - game_start.mp3   → 1.0 (full)
   - explosion.mp3    → 1.0 (full)
   - freeze.mp3       → 0.8
   - Page transition  → 0.3 (very subtle)
   - Bomb tick        → 0.7

5. Always check if sound is enabled in settings
   before playing — respect the mute toggle

6. Never play two copies of the same file simultaneously
   — cancel the previous play before restarting

7. Preload all sounds in initState of the app root
   so there is zero delay on first play
```
