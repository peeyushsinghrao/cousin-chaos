### FIX 3A — Settings tab broken (pushes route instead of switching IndexedStack)

**Problem:** The Settings nav item in `_buildBottomNavBar()` calls `Navigator.push(slideUpRoute(SettingsScreen()))` instead of `setState(() => _selectedTab = 3)`. This means:
- The Settings screen is NOT in the `IndexedStack` (only 3 children: Chaos, Crew, Vault)
- The tab index 3 is selected visually but nothing renders in the stack
- Navigating to Settings and back loses the tab state

**Fix — two changes:**

**Step 1:** Add `SettingsScreen` as the 4th child of `IndexedStack`:
```dart
// Add this import at top if not present:
import '../settings/settings_screen.dart';

// In _buildGameModeGrid's parent IndexedStack, change children from:
children: [
  SingleChildScrollView(...),   // index 0 — Chaos
  const CrewScreen(),           // index 1 — Crew
  const VaultScreen(),          // index 2 — Vault
],

// To:
children: [
  SingleChildScrollView(...),   // index 0 — Chaos
  const CrewScreen(),           // index 1 — Crew
  const VaultScreen(),          // index 2 — Vault
  const SettingsScreen(),       // index 3 — Settings
],
```

**Step 2:** Change the Settings nav item's `onTap` from `Navigator.push` to `setState`:
```dart
// BEFORE:
_buildNavItem(
  icon: Icons.settings,
  label: 'Settings',
  isSelected: _selectedTab == 3,
  onTap: () {
    Navigator.push(context, slideUpRoute(const SettingsScreen()));
  },
),

// AFTER:
_buildNavItem(
  icon: Icons.settings,
  label: 'Settings',
  isSelected: _selectedTab == 3,
  onTap: () => setState(() => _selectedTab = 3),
),
```

## PHASE 4 — VAULT SCREEN FIX

**File:** `lib/screens/vault/vault_screen.dart`

### FIX 4A — `Expanded` inside `SingleChildScrollView` render error

**Problem:** `vault_screen.dart` wraps content in `SingleChildScrollView` but uses `Expanded` widgets inside a `Column` within it (lines 80, 115, 117, 126, 128, 193). `Expanded` inside `SingleChildScrollView` causes a Flutter render error at runtime because `SingleChildScrollView` provides unbounded height.

**Fix:** Replace every `Expanded(child: X)` inside the scrollable column with `SizedBox(width: double.infinity, child: X)` or `IntrinsicHeight(child: X)` depending on context.

Specifically:
- Line 80: `Expanded(child: _buildMiniSessionCard(...))` inside a `Row` inside `GlassCard` inside the `SingleChildScrollView`. Rows can use `Expanded` fine — leave those.
- Lines 115, 117: `Expanded(child: _buildMiniSessionCard(...))` inside a `Row` — these are fine, leave them.
- Lines 126, 128: `Expanded(child: _buildStatCard(...))` inside a `Row` — fine, leave them.
- Line 193: `Expanded(child: Column(...))` inside a `Row` inside `GlassCard` — also fine if in a Row.

Read the file fully before editing. Only remove `Expanded` that is a **direct child of a Column** (not a Row) that itself is inside `SingleChildScrollView`. If `Expanded` is directly inside a Column (not Row), replace with `SizedBox(width: double.infinity, ...)`.
